# 🚀 Настройка GitHub Pages для React приложения

## 📋 Текущая ситуация

У вас уже настроены GitHub Pages, но они деплоятся из ветки `main` вместо папки `frontend/dist`. Нужно изменить настройки.

## 🔧 Пошаговая настройка

### Шаг 1: Измените источник деплоя

1. Перейдите в ваш репозиторий: https://github.com/komanyak/rasp
2. Откройте **Settings** → **Pages**
3. В разделе **Source** выберите:
   - **Source**: `Deploy from a branch`
   - **Branch**: `gh-pages`
   - **Folder**: `/ (root)`
4. Нажмите **Save**

### Шаг 2: Дождитесь создания ветки gh-pages

После первого запуска workflow `deploy.yml` автоматически создастся ветка `gh-pages` с собранным приложением.

### Шаг 3: Проверьте деплой

1. Подождите 2-3 минуты после настройки
2. Откройте https://komanyak.github.io/rasp/
3. Должно отображаться ваше React приложение

## 🤖 Автоматический деплой

### Как это работает:

1. **При изменении файлов в `frontend/`** → автоматически запускается `deploy.yml`
2. **При обновлении расписания** → `update-schedule.yml` обновляет данные и коммитит изменения
3. **Коммит изменений** → автоматически запускается `deploy.yml`

### Workflow файлы:

- **`deploy.yml`** - деплоит фронтенд на GitHub Pages
- **`update-schedule.yml`** - обновляет данные расписания

## 📊 Мониторинг

### Проверка статуса:
1. **Actions** → **Deploy to GitHub Pages** - статус деплоя
2. **Actions** → **Update Schedule Data** - статус обновления данных

### Что должно работать:
- ✅ Сайт доступен по адресу https://komanyak.github.io/rasp/
- ✅ Отображается информация о последнем обновлении
- ✅ Работает выбор института и группы
- ✅ Скачивание файлов .ics

## 🔍 Устранение неполадок

### Проблема: Сайт не обновляется
**Решение:**
- Проверьте статус в Actions
- Убедитесь, что ветка `gh-pages` создалась
- Проверьте настройки Pages в Settings

### Проблема: 404 ошибка
**Решение:**
- Убедитесь, что в Settings → Pages выбран branch `gh-pages`
- Подождите несколько минут после настройки

### Проблема: Старый контент отображается
**Решение:**
- Очистите кеш браузера (Ctrl+F5)
- Проверьте, что последний коммит в `gh-pages` актуальный

## 🎯 Ручной деплой

Если нужно принудительно пересобрать и задеплоить:

1. **Через GitHub Actions:**
   - Actions → Deploy to GitHub Pages → Run workflow

2. **Локально:**
   ```bash
   cd frontend
   npm run build
   npm run deploy
   ```
