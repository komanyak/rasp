# 🌐 Настройка GitHub Pages

## ✅ Шаг 1: Код уже загружен в GitHub

Ваш код уже успешно загружен в репозиторий: https://github.com/komanyak/rasp

## 🔧 Шаг 2: Настройка GitHub Pages

### Перейдите в настройки репозитория:

1. **Откройте ваш репозиторий**: https://github.com/komanyak/rasp
2. **Нажмите на вкладку "Settings"** (в верхнем меню)
3. **Прокрутите вниз до раздела "Pages"** (в левом меню)

### Настройте источник:

1. **В разделе "Source"** выберите: **"Deploy from a branch"**
2. **В разделе "Branch"** выберите: **"gh-pages"**
3. **В разделе "Folder"** выберите: **"/ (root)"**
4. **Нажмите кнопку "Save"**

## ⏳ Шаг 3: Ожидание деплоя

- GitHub Pages автоматически начнет деплой
- Обычно это занимает **1-5 минут**
- Статус можно отслеживать в разделе **"Actions"**

## 🎯 Шаг 4: Проверка результата

После завершения деплоя ваш сайт будет доступен по адресу:
**https://komanyak.github.io/rasp**

## 🔍 Шаг 5: Проверка работы

### Что должно работать:
- ✅ Красивый интерфейс с градиентным фоном
- ✅ Выпадающие списки институтов, курсов, групп
- ✅ Информация о последнем обновлении
- ✅ Скачивание файлов .ics
- ✅ Адаптивный дизайн на мобильных

### Тестирование:
1. **Откройте сайт**: https://komanyak.github.io/rasp
2. **Выберите институт** (например, "Институт №3")
3. **Выберите курс** (например, "1 курс")
4. **Выберите уровень** (например, "Бакалавриат")
5. **Выберите группу** (например, "М3О-101Б-23")
6. **Нажмите "Скачать расписание"**
7. **Проверьте, что файл скачался**

## 🤖 Шаг 6: Настройка GitHub Actions

### Включение автоматических обновлений:

1. **В настройках репозитория** → **Actions** → **General**
2. **Actions permissions**: ✅ "Allow all actions and reusable workflows"
3. **Workflow permissions**: ✅ "Read and write permissions"

### Проверка workflow:
1. **Перейдите в раздел "Actions"**
2. **Выберите "Update Schedule Data"**
3. **Нажмите "Run workflow"** для первого запуска

## 📊 Шаг 7: Мониторинг

### Отслеживание обновлений:
- **Actions** → показывает статус автоматических обновлений
- **На сайте** → отображается время последнего обновления
- **Коммиты** → автоматические коммиты с обновлениями

### Уведомления:
- GitHub будет отправлять email о статусе workflow
- Можно настроить в **Settings** → **Notifications**

## 🚨 Устранение проблем

### Проблема: Сайт не открывается
**Решение:**
- Подождите 5-10 минут после настройки Pages
- Проверьте, что branch "gh-pages" создался
- Убедитесь, что в настройках выбран правильный branch

### Проблема: GitHub Actions не запускается
**Решение:**
- Проверьте настройки Actions в Settings
- Убедитесь, что workflow файл в `.github/workflows/`
- Проверьте права доступа

### Проблема: Файлы не скачиваются
**Решение:**
- Проверьте, что файлы .ics находятся в `frontend/public/rasp/`
- Убедитесь, что GitHub Actions обновил данные
