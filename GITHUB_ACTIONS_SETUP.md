# 🚀 Настройка GitHub Actions для автоматического обновления

## 📋 Предварительные требования

1. **Репозиторий на GitHub** (уже есть: https://github.com/komanyak/rasp)
2. **Настроенные GitHub Pages** (инструкция в `frontend/DEPLOYMENT.md`)
3. **Права на запись в репозиторий**

## 🔧 Настройка GitHub Actions

### Шаг 1: Включение GitHub Actions

1. Перейдите в ваш репозиторий на GitHub
2. Откройте вкладку **Settings**
3. В левом меню найдите **Actions** → **General**
4. Убедитесь, что **Actions permissions** установлены:
   - ✅ **Allow all actions and reusable workflows**
   - ✅ **Allow GitHub Actions to create and approve pull requests**

### Шаг 2: Настройка GitHub Pages

1. В **Settings** → **Pages**
2. **Source**: Deploy from a branch
3. **Branch**: gh-pages
4. **Folder**: / (root)
5. Нажмите **Save**

### Шаг 3: Проверка workflow файла

Убедитесь, что файл `.github/workflows/update-schedule.yml` существует в репозитории.

## 🎯 Первый запуск

### Автоматический запуск:
- Workflow запустится автоматически в 6:00 UTC следующего дня

### Ручной запуск:
1. Перейдите в **Actions** в вашем репозитории
2. Выберите **Update Schedule Data**
3. Нажмите **Run workflow**
4. Выберите опции:
   - **Force update**: `false`
5. Нажмите **Run workflow**

## 📊 Мониторинг выполнения

### Где смотреть:
1. **Actions** → **Update Schedule Data** → [номер запуска]
2. Вкладка **Summary** - общая информация
3. Вкладка **update-schedule** - детальные логи

### Что проверять:
- ✅ **Checkout repository** - успешно
- ✅ **Set up Python** - успешно
- ✅ **Install dependencies** - успешно
- ✅ **Run schedule update script** - успешно
- ✅ **Check for changes** - показывает статус изменений
- ✅ **Commit and push changes** - коммитит изменения
- ✅ **Deploy to GitHub Pages** - деплоит сайт

## 🔍 Проверка результатов

### После успешного выполнения:

1. **Проверьте коммит**:
   ```bash
   git log --oneline -5
   ```
   Должен появиться коммит вида: "🔄 Auto-update schedule data..."

2. **Проверьте файлы**:
   - `frontend/public/groups.json` - обновлен
   - `frontend/public/rasp/*.ics` - обновлены
   - `frontend/public/update-info.json` - создан/обновлен

3. **Проверьте сайт**:
   - Откройте ваш сайт на GitHub Pages
   - В заголовке должно отображаться время последнего обновления

## ⚠️ Устранение неполадок

### Проблема: Workflow не запускается
**Решение:**
- Проверьте настройки Actions в Settings
- Убедитесь, что файл workflow находится в `.github/workflows/`

### Проблема: Ошибка "Permission denied"
**Решение:**
- Проверьте, что у `GITHUB_TOKEN` есть права на запись
- В Settings → Actions → General → Workflow permissions должно быть "Read and write permissions"

### Проблема: Ошибка парсинга
**Решение:**
- Проверьте доступность сайта МАИ
- Посмотрите логи в GitHub Actions для деталей

### Проблема: Деплой не работает
**Решение:**
- Проверьте настройки GitHub Pages
- Убедитесь, что branch `gh-pages` создался
- Проверьте, что в корне есть файл `index.html`

## 📅 Расписание обновлений

### Автоматические обновления:
- **Время**: Каждый день в 6:00 UTC (9:00 МСК)
- **Часовой пояс**: UTC
- **Частота**: Ежедневно

### Ручные обновления:
- Доступны в любое время
- Через интерфейс GitHub Actions
- С опцией принудительного обновления

## 🔔 Уведомления

### GitHub уведомления:
- На email приходят уведомления о статусе workflow
- Можно настроить в Settings → Notifications

### Мониторинг:
- Регулярно проверяйте раздел Actions
- При ошибках workflow покажет детальную информацию

## 📈 Статистика

После каждого обновления в Summary отображается:
- Количество измененных файлов
- Время выполнения
- Статус каждого этапа
- Информация о последнем обновлении
